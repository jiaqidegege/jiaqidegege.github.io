由图片预加载引出的浏览器渲染问题

标签（空格分隔）： 未分类

一，先说结论：

1.css文件的影响

css不会阻塞HTML的解析但是会阻塞渲染
css放在head是为了页面获得样式后再渲染
为了加快首屏出现的时间，可以把和首屏无关的样式放在尾部

2.js文件的影响

外部js放在首部可能导致HTML截断，也就是css已经下载好了开始渲染页面，但是后面的HTML解析被如果被可能很慢的js阻塞了,所以就更不用说渲染出来了。
js（内联）放在首部还可能导致请求到空元素.

3.图片资源的影响

图片会让页面的重绘，但不会影响HTML解析与页面渲染

二，浏览器的渲染过程

  1. Create/Update DOM And request css/image/js：浏览器请求到HTML代码后，在生成DOM的最开始阶段（应该是 Bytes → characters 后），并行发起css、图片、js的请求，无论他们是否在HEAD里。
    注意：发起 js 文件的下载 request 并不需要 DOM 处理到那个 script 节点，比如：简单的正则匹配就能做到这一点，虽然实际上并不一定是通过正则：）。这是很多人在理解渲染机制的时候存在的误区。
  2. Create/Update Render CSSOM：CSS文件下载完成，开始构建CSSOM
  3. Create/Update Render Tree：所有CSS文件下载完成，CSSOM构建结束后，和 DOM 一起生成 Render Tree。
  4. Layout：有了Render Tree，浏览器已经能知道网页中有哪些节点、各个节点的CSS定义以及他们的从属关系。下一步操作称之为Layout，顾名思义就是计算出每个节点在屏幕中的位置。
  5. Painting：Layout后，浏览器已经知道了哪些节点要显示（which nodes are visible）、每个节点的CSS属性是什么（their computed styles）、每个节点在屏幕中的位置是哪里（geometry）。就进入了最后一步：Painting，按照算出来的规则，通过显卡，把内容画到屏幕上。

三，关于重排（reflow）和重绘(repaint)

  1. 触发条件
    凡是会改变元素位置大小的操作都会引起重排，重排一定会引发重绘
    凡是改变元素“外表”的操作都会引起重绘，重绘不一定引起重排
  2. 减少重排和重绘以提高性能

    四，图片预加载的应用场景

    1.图片预加载就是为了在空闲的时间里对将要用到（暂时不会下载，只有用到时（用户事件比如点击控制）再下载）的图片提前加载到缓存中
    2.或者对于一个图片很多的页面先进行遮挡（比如进度条），等（用脚本控制）统一下载完在进行显示，防止因各张图片参差不齐造成不好的用户体验

1.HTML Questions

doctype 的用途

doctype 是 document type 的缩写。它被声明用来区分标准模式和怪异模式，也就是告诉浏览器该以怎样的模式去解析和渲染网页。在HTML5中，DOCTYPE的唯一作用就是启用标准模式

标准模式 、接近标准模式 、怪异模式的区别

标准模式： 完全遵循HTML 和 css 规范进行布局
怪异模式： 布局模拟Netscape Navigator 4 和Internet Explorer 5中的非标准行为
接近标准模式： 有少量的特性是以怪异模式实现的，其他都遵循标准规范

HTML 和 XHTML 有什么区别

XHTML是用XML的规则重新规范HTML，XHTML是XML的一个子集。XHTML对格式的要求比HTML更为严格：

  • XHTML 要求正确嵌套
  • XHTML 所有元素必须被正确关闭
  • XHTML 区分大小写且元素名必须小写
  • XHTML 属性值要用双引号且不能为空
  • XHTML 在元素内容里不能有尖括号和&，需要转义

以application/xhtml+xml 提供页面会出现哪些问题

由于XHTML/xml的语法规范比较严格，凡是没有完全遵循XML规范的页面都不会得到正常解析，导致用户得到无法阅读的页面，特别的，在IE中，声明application/xhtml+xml的页面会导致在IE8中显示一个未知格式的下载框，因为第一个支持XML的IE版本是IE9····

什么是data- 属性？

在前端框架变得流行之前，data-属性是作为自定义属性常用来存储一些页面数据。但是现在已经不鼓励使用，一是因为用户可以通过浏览器的审查元素轻松修改数据属性，二是现在有更好的解决方案————数据绑定。比如Vue中的双向绑定

HTML5相比上一版本有哪些变化要点

  • 语义化 让你更准确的描述网页内容
  • 离线浏览和客户端存储 更好的解决方案用于网页在客户端浏览器上存储数据以及更高效的离线操作方式
  • 多媒体支持 让多媒体在网页上变得更加便于展示
  • 2D/3D的图表和效果 使得展示效果变得更加多样化和绚丽

描述cookie、localStorage、sessionStorage的区别

描述<script>、<script async>、<script defer>的区别

<script>标签会阻塞HTML解析，也就是说当页面HTML解析碰到script标签时，会停下来，并去获取脚本且立即执行，执行完毕后会恢复HTML解析
异步脚本的获取是和HTML解析并行执行的，一旦脚本获取完毕就立即执行，所以，异步脚本可能会在页面解析完成前执行完毕，因此，异步脚本不应依赖其他脚本。
延迟脚本的获取同样是和HTML解析并行执行，但执行却是在页面解析完成后，这样做适用于某些依赖于获取页面解析完成后的dom元素的脚本。延迟脚本不能包含document.write语句，当页面有多个延迟脚本时，按在页面的放置顺序执行，延迟脚本和将普通脚本放置在body标签后没有区别

为什么一般将css link放在head中，而将script放在尾部

link标签放在head中是规范的一部分，而且放在head中允许页面进行逐行渲染，这样能提高用户体验，如果放在尾部，在某些浏览器中可能会导致白屏，因为在样式表下载完成之前一些浏览器会阻止渲染以防止重绘。将link放在head中还可以防止文档样式闪烁。
script标签因为会阻塞HTML解析，因此常放在尾部。现在也可以使用放在头部的延迟脚本达到同样效果

全面了解JavaScript中的对象

标签（空格分隔）： JavaScript 对象

一认识ECMAScript内置对象，本地对象，宿主对象

1.定义

本地对象官方定义：独立于宿主对象，由ECMAscript实现提供的对象
内置对象官方定义：独立于宿主对象，由ECMAScript实现提供的对象，在ECMASript开始执行时创建
宿主对象官方定义：由ECMAScript实现的宿主环境提供的对象

比较抽象，翻译一下：

本地对象：就是JavaScript中的所有引用类型，包括
Number String Boolean Array Date Math Functions RegExp Events Error等
内置对象：Math Global
Global对象是所有全局变量和函数的占位符，它不是任何对象的属性，所以没有名称。在具体的环境中，比如在客户端浏览器中指的是window，node中是global。
global对象的属性包括JavaScript预先定义的一些顶层属性和函数，以及用户在顶层作用域中自定义的一些属性和函数
全局对象构成了全局作用域
宿主对象：window document 以及用户自定义的对象

2.关系与区别

描述多个对象之间的关系和区别总是显得无比杂乱。也没有好的解决办法，在这里采用几个命题和一个图进行描述。

  • 内置对象是本地对象的子集
  • 内置对象在引擎初始化阶段就被创建（不需要实例化，直接调用），本地对象是在程序运行过程中动态创建（需要实例化）
  • 内置对象是一个对象，不是类，它既没有构造函数（new Math（）），也无法实例化一个新的内置对象

注：在JavaScript中，所有本地对象包括内置对象统一叫做标准（内置）对象。

JavaScript模块化开发

标签（空格分隔）： morder

今天在学习jQuery插件封装的时候，了解了一下JavaScript的模块化开发，现整理如下：

1. 什么是JavaScript模块化开发？

模块化开发是一种代码组织方式，用来管理组织越来越庞大复杂的代码之间的关系；

2. 模块化是做什么用的，或者说解决什么问题？

  • 命名冲突：当项目由团队进行协作开发的时候，不同开发人员的变量和函数命名可能相同；即使是一个开发，当开发周期比较长的时候，也有可能会忘记之前使用了什么变量，从而导致重复命名，导致命名冲突
  • 文件依赖：代码重用产生了文件之间的依赖关系，比如引入文件的顺序，数目，当项目变得庞大时，这些会变得越来越难以维护。

3. 一个模块化系统应该实现什么功能？

在了解了模块化的作用后，不难发现，我们要想实现一个模块化系统，应当具有以下三个功能：

  • 创建模块
  • 导入模块/导入依赖
  • 导出模块/定义外部接口

4.js模块的规范

模块化要想起作用，还得有一个统一的开发规范，要不然就没有实际意义了，就像大家如果都说方言，谁都听不懂谁，所以有了普通话。一个统一的规范有利于可交换性和可移植性，现在被广泛支持的主要有两种规范：
commonjs 和 AMD。

commonjs

    experts.a = function(){}

AMD

AMD定义了如下接口：

  1. 一个全局性的define方法。API如下：
    define（id？，dependencies?, factory）
  • id string类型 用来定义模块名，一般省略
  • depenencies Array类型 用于定义依赖模块，成员是模块名
  • factory function类型 一个回调函数，在依赖的模块加载成功后，会执行这个回调函数，它的参数是所有依赖模块的引用，如果回调函数有返回值，会导出出来

2. 一个本地代码库操作流程示范

  • 创建一个文件夹test: mkdir test
  • 切换到创建的目录test: cd test
  • 将该目录初始化为一个git仓库: git init
  • 新建一个a.md文件: touch a.md
  • 将a.md加入git仓库的缓存: git add a.md
  • //将缓存中的a.md删除:git rm cached
  • 将a.md正式提交到仓库 -m代表提交信息: git commit -m 'first commit'
  • 查看仓库状态信息，比如有什么文件待缓存，待提交之类的: git states
  • 查看commit信息 可以看到提交人，提交人邮箱 ，提交信息: git log
  • 创建当前分支（master）的一个分支a:git branch a
  • 将当前工作分支切换到分支a:git checkout a
  • //上述两个命令可以合并成: git checkout -b a
  • 为当前分支创建一个v1.0标签git tag v1.0
  • 切换到标签为v2.0的代码:git checkout v2.0
  • 将当前分支合并到主干分支master:git merge
  • 删除当前分支:git branch -d
  • 强制删除当前分支，用于当前分支还有未合并的改动的情况:git branch -D
