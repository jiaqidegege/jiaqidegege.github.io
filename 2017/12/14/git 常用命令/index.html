2. 一个本地代码库操作流程示范

  • 创建一个文件夹test: mkdir test
  • 切换到创建的目录test: cd test
  • 将该目录初始化为一个git仓库: git init
  • 新建一个a.md文件: touch a.md
  • 将a.md加入git仓库的缓存: git add a.md
  • //将缓存中的a.md删除:git rm cached
  • 将a.md正式提交到仓库 -m代表提交信息: git commit -m 'first commit'
  • 查看仓库状态信息，比如有什么文件待缓存，待提交之类的: git states
  • 查看commit信息 可以看到提交人，提交人邮箱 ，提交信息: git log
  • 创建当前分支（master）的一个分支a:git branch a
  • 将当前工作分支切换到分支a:git checkout a
  • //上述两个命令可以合并成: git checkout -b a
  • 为当前分支创建一个v1.0标签git tag v1.0
  • 切换到标签为v2.0的代码:git checkout v2.0
  • 将当前分支合并到主干分支master:git merge
  • 删除当前分支:git branch -d
  • 强制删除当前分支，用于当前分支还有未合并的改动的情况:git branch -D
