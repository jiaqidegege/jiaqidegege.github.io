1.HTML Questions

doctype 的用途

doctype 是 document type 的缩写。它被声明用来区分标准模式和怪异模式，也就是告诉浏览器该以怎样的模式去解析和渲染网页。在HTML5中，DOCTYPE的唯一作用就是启用标准模式

标准模式 、接近标准模式 、怪异模式的区别

标准模式： 完全遵循HTML 和 css 规范进行布局
怪异模式： 布局模拟Netscape Navigator 4 和Internet Explorer 5中的非标准行为
接近标准模式： 有少量的特性是以怪异模式实现的，其他都遵循标准规范

HTML 和 XHTML 有什么区别

XHTML是用XML的规则重新规范HTML，XHTML是XML的一个子集。XHTML对格式的要求比HTML更为严格：

  • XHTML 要求正确嵌套
  • XHTML 所有元素必须被正确关闭
  • XHTML 区分大小写且元素名必须小写
  • XHTML 属性值要用双引号且不能为空
  • XHTML 在元素内容里不能有尖括号和&，需要转义

以application/xhtml+xml 提供页面会出现哪些问题

由于XHTML/xml的语法规范比较严格，凡是没有完全遵循XML规范的页面都不会得到正常解析，导致用户得到无法阅读的页面，特别的，在IE中，声明application/xhtml+xml的页面会导致在IE8中显示一个未知格式的下载框，因为第一个支持XML的IE版本是IE9····

什么是data- 属性？

在前端框架变得流行之前，data-属性是作为自定义属性常用来存储一些页面数据。但是现在已经不鼓励使用，一是因为用户可以通过浏览器的审查元素轻松修改数据属性，二是现在有更好的解决方案————数据绑定。比如Vue中的双向绑定

HTML5相比上一版本有哪些变化要点

  • 语义化 让你更准确的描述网页内容
  • 离线浏览和客户端存储 更好的解决方案用于网页在客户端浏览器上存储数据以及更高效的离线操作方式
  • 多媒体支持 让多媒体在网页上变得更加便于展示
  • 2D/3D的图表和效果 使得展示效果变得更加多样化和绚丽

描述<script>、<script async>、<script defer>的区别

<script>标签会阻塞HTML解析，也就是说当页面HTML解析碰到script标签时，会停下来，并去获取脚本且立即执行，执行完毕后会恢复HTML解析
异步脚本的获取是和HTML解析并行执行的，一旦脚本获取完毕就立即执行，所以，异步脚本可能会在页面解析完成前执行完毕，因此，异步脚本不应依赖其他脚本。
延迟脚本的获取同样是和HTML解析并行执行，但执行却是在页面解析完成后，这样做适用于某些依赖于获取页面解析完成后的dom元素的脚本。延迟脚本不能包含document.write语句，当页面有多个延迟脚本时，按在页面的放置顺序执行，延迟脚本和将普通脚本放置在body标签后没有区别

为什么一般将css link放在head中，而将script放在尾部

link标签放在head中是规范的一部分，而且放在head中允许页面进行逐行渲染，这样能提高用户体验，如果放在尾部，在某些浏览器中可能会导致白屏，因为在样式表下载完成之前一些浏览器会阻止渲染以防止重绘。将link放在head中还可以防止文档样式闪烁。
script标签因为会阻塞HTML解析，因此常放在尾部。现在也可以使用放在头部的延迟脚本达到同样效果

2.CSS Questions

关于选择器的优先级及权重计算

从CSS样式位置看优先级 ：内嵌样式 > 内联样式 > 外部样式
从选择器类型看优先级：important > 内嵌样式（权值1000） > id选择器（权值100） > 类、伪类、属性选择器（10） > 元素、伪元素选择器（1）
